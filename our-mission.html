Our Mission

How public code works

Public code is open source software fulfilling the essential roles of public organizations. Through adaptive re-use, a network of public administrations can contribute back to the software, so its development and maintenance become truly collaborative.
Being open unlocks many other things. Local responsibility and democratic accountability are ensured when organizations implement and maintain their own public code. Openness with a broad contributor base creates more secure software, with many eyes spotting potential flaws. Those contributors share in maintenance work, keeping it functional and modern, reducing future technical debt, and making it more sustainable over time. Openness makes code and data more easily adaptable to regional needs, and as those needs change—the code becomes easier to retool, repurpose or retire. All of this reduces risk in public infrastructure.
Ultimately, this open pooling of resources lets public administrations focus on optimizing software for local contexts, creating better user experiences for their end users—residents and citizens.

We think there are three core problems that speak to the need for public code:

1

The software running our society—and the data we generate—is often hidden in proprietary, privately-controlled codebases. These black boxes undermine the integrity of government.

2

To handle an emerging array of challenges—from public health to extreme climate events—public administrations must increasingly rely an software they are ill-equipped to build and deploy.

3

Governments typically acquire software through isolated purchases, and are then subject to ongoing license fees and vendors’ cost increases. Their capacity to manage, maintain and control software decays over time.

The powerful alternative—open source software—has inherent challenges:

Discovery

Finding fit-for-purpose, open, adoptable software

Flexibility

Enabling future replication
in new contexts

Community

The critical mass of developers needed to collaboratively maintain open software

Co-procurement

Pooling resources to fund custom development and maintenance

We believe a third party can help governments overcome the challenges of open source software. That third party is built around communities of practice that work collaboratively, stewarding the code running our public infrastructure follows the best practices we have coalesced into The Standard for Public Code.
